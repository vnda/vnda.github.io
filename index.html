Sobre o liquid

Liquid é a linguagem de programação é responsável por implementar os templates desenvolvidos e utilizados pela Vnda, processando os arquivos modelo .liquid

Templates

Os templates ficam na raiz da pasta compartilhada no Dropbox onde todos os arquivo são salvos na extensão .liquid

Dentro da raiz ficam os diretórios "images" "stylesheets" "javascript" e "page"

Dentro do diretórios page/ é gerado os arquivos de extensão .html das página que são criados no admin

Enviar Produto ao Carrinho

Para adicionar itens ao carrinho, você precisa fazer um POST para a URL definida na variável {{ cart_url }}, passando dois campos:

* product_id: ID do produto selecionado pelo usuário
* quantity: Quantidade de itens que o usuário deseja comprar

Os campos product_id e quantity são obrigatório, mesmo que o produto tenha apenas uma variante. Caso sua loja não permita que o usuário compre mais de um item, você deve informar este campo com o valor 1. 

// deprecate, atributos modelo antigo
<form action="{{ cart_url }}" method="POST">
  <input type="hidden" name="product_id" value="{{ product.id }}"/>
  <input type="number" name="quantity" min="1"/>
  <input type="submit" value="Adicionar"/>
</form>

Isso pode ser feito utilizando um formulário »


// deprecate, atributos modelo antigo
<form action="{{ cart_url }}" method="POST">
  <input type="hidden" name="product_id" value="{{ product.id }}"/>
  <input type="number" name="quantity" min="1"/>
  <select name="attributes[tamanho]">
    <option value="P">P</option>
    <option value="M">M</option>
    <option value="G">G</option>
  </select>
  <input type="submit" value="Adicionar"/>
</form>

Em caso de o produto ter variantes » 


// deprecate, atributos com modelo antigo
$.post("{{ cart_url }}", { variant_id: 999, quantity: 1 }, function() {
  alert("Item adicionado com sucesso");
});

OU via AJAX »


Caso o produto possua variantes, você deve enviá-las junto com os parâmetros obrigatórios, da seguinte forma:

* attributes[cor]: Valor do atributo cor da variante
* attributes[tamanho]: Valor do atributo tamanho da variante

E assim para cada atributo configurado no admin.

// deprecate, atributos com modelo antigo

Além disso, você pode adicionar informações extras a cada item do carrinho. Essas informações podem ser, por exemplo, uma observação do cliente ou ainda uma imagem para ser impressa no produto. Para enviar essas informações, basta criar os campos a seguinte forma »

Atualizar item do carrinho

<form action="{{ cart_url }}" method="post">
  <input type="hidden" name="_method" value="patch" />
  <input type="hidden" name="item_id" value="999" />
  <input type="hidden" name="quantity" value="1" />
  <input type="submit" value="Alterar"/>
</form>

Para alterar a quantidade de item do carrinho, você precisa fazer um POST para a URL definida na variável {{ cart_url }}

Remover item do carrinho

<form action="{{ cart_url }}" method="post">
  <input type="hidden" name="_method" value="patch" />
  <input type="hidden" name="item_id" value="999" />
  <input type="submit" value="Excluir"/>
</form>

Para Remover um produto do carrinho, você precisa fazer um POST para a URL definida na variável {{ cart_url }}, passando dois campos:
* item_id: Id do item selecionado pelo usuário
* _method: Informar o value "Delete"


$.post("{{ cart_url }}", { item_id: 999, cart_method: "Delete" }, function() {
  alert("Produto Removido com sucesso");
});

A plataforma Vnda permite que pedidos sejam feitos tanto para pessoas físicas, quanto jurídicas. O que define quando é um ou outro é o campo documents, se ele possuir a chave cpf e for um CPF válido, então é pessoa física, se ele possuir a chave cnpj e for um CNPJ válido, é pessoa júridica.
