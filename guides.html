¶ Design

Sugerimos a seguinte ordem para criação das telas para os layout de um canal ecommerce que utilize a plataforma Vnda:

  • Home: tela inicial do site, pode exibir produtos, banners, entre outras coisas. Esta tela deve considerar, caso exista, uma estrutura de header e footer utilizada em toco o canal.

  • Categorias (tags): tela de "categoria", exibe uma lista de produtos que estão vinculados a tag. Essa página também pode servir para exibir "sub-categorias".

  • Produto: tela que exibe os detalhes dos produtos. Aqui ele pode selecionar as variantes e adicionar a variante selecionada ao carrinho.

  • Carrinho: tela de carrinho, onde ficam listados os produtos comprados, o resumo da compra e onde o cliente faz o cálculo de frete, informando o CEP para entrega. Nesta página também fica o form onde o cliente seleciona o meio de entrega após informar o CEP.

  • Pedido: tela com o form no qual o cliente preenche com os dados de entrega do pedido. Geralmente essa página vem depois do cliente ter selecionado a meio de entrega (**cart.erb**)

  • Pagamento: tela onde serão exibidas para o cliente as formas de pagamento disponíveis no site. Aqui ele seleciona uma delas e informa os dados necessários para pagar pelo compra. Geralmente essa página vem depois do cliente informar os dados de entrega do pedido (**order.erb**).

  • Concluído Cartão: tela de conclusão da compra, onde é exibida a mensagem que a compra foi efetuada com sucesso.

  • Concluído Boleto: tela de conclusão da compra, onde é exibida a mensagem que a compra foi efetuada com sucesso deve conter o link para o impressão do boleto.

  • Página: através deste template serão exibidas as páginas estáticas criadas no admin.

  • Busca: tela que exibe o resultado da busca. Basicamente uma lista de produtos que batem com o termo de busca informado pelo cliente.

  • Login: tela onde fica o form de login e signup e Facebook Connect.

  • Conta: tela onde são exibidos os dados da conta do cliente, onde ele consulta os pedidos e os pontos adquiridos atráves do indicações do site (se for o caso). Nesta tela também deve ser usada para o form de alteração de nome, email e senha do cliente.

  • Erro 404: HTML que será exibido quando uma página não for encontrada. Ele precisa replicar o header e o footer do site.

¶ Sobre o liquid

Liquid é a linguagem de programação é responsável por implementar os templates desenvolvidos e utilizados pela Vnda, processando os arquivos modelo .liquid

Como Usar ler sobre.

¶ Templates

Os templates ficam na raiz da pasta compartilhada no Dropbox onde todos os arquivo são salvos na extensão .liquid

Dentro da raiz ficam os diretórios "images" "stylesheets" "javascript" e "page"

Dentro do diretórios page/ é gerado os arquivos de extensão .html das página que são criados no admin

Vejas os templates disponíveis na plataforma Iniciar

¶ Layout.liquid

Neste template ficam o header e o footer do site, é onde as partes comuns do HTML são colocadas para serem utilizadas por todos os templates (exceto 404 e facebook).

¶ Home.liquid

Página inicial da loja, aqui podem ser exibidos produtos em destaque, banners, menus e qualquer outra informação que for necessária para atrair o cliente a navegar no resto do site.

¶ Tag.liquid

Página que exibe produtos relacionados a uma tag. Tag é a forma de categorização utilizada pela Vnda para organizar os produtos.

¶ Product.liquid

Template para exibição do produto, suas imagens, produtos relacionados. Aqui também fica o formulário que permite adicionar o item ao carrinho, neste formulário o cliente pode selecionar qual variante do produto (cor, tamanho, etc) ele realmente quer. 

          

      

        

        

            
 ¶ Cart.liquid 

            

              

                
 1ª etapa do processo de compra. No template do carrinho são exibidos os itens já adicionados e onde pode ser feito o cálculo do frete. Pode exibir também um resumo do total do pedido. 

              

            

          

        
      

        

        

            
 ¶ Address.liquid 

            

              

                
 Template onde é exibido o formulário responsável por capturar as informações para entrega do pedido, assim como dados pessoais do cliente, como e-mail e CPF. 

              

            

          

        
      

        

        

            
 ¶ Payment.liquid 

            

              

                
 3ª etapa do processo de compra. Aqui o cliente informa os dados do cartão de crédito ou dá inicio a geração do boleto. 

              

            

          

        
      

        

        

            
 ¶ Finished.liquid 

            

              

                
 4ª e última etapa do processo de compra. Aqui são exibidas as informações do pedido, como o número e/ou o link para impressão do boleto, junto com a mensagem de conclusão da compra. 

              

            

          

        
      

        

        

            
 ¶ Account.liquid 

            

              

                
 Template usado para exibir os dados da conta do cliente, os pedidos efetuados, assim como a situação deles. Nesta página o cliente também pode alterar os dados pessoais, como nome, e-mail e senha. 

              

            

          

        
      

        

        

            
 ¶ Login.liquid 

            

              

                
 Template usado tanto para novos cadastros quanto para logins de clientes já cadastrados. Desta página parte também o login via Facebook, que permite que clientes participem de promoções específicas.

 Página que exibe o resultado de uma busca, este resultado é uma lista de produtos que fazem match com o termo buscado pelo cliente atráves da URL /busca?q=termo.

 ¶Como Usar o Liquid?

            

              

                
Existem dois tipos de marcação no líquido:

                
Tags de Lógica: são declarações condicionais. Eles são a afirmação de que diz o código e o que fazer.

                
Tags de saída: são as etiquetas de saída exibir o líquido encontrado entre ele.
                  Você também pode usar filtros para alterar a saída retornado.

                
Para usar o líquido, você vai precisar de recorrer a uma combinação de lógica e de saída de etiquetas usando operadores líquidos. Acções com liquidez muitos dos mesmos operadores como outras linguagens de programação:

 ¶ Enviar Produto ao Carrinho ( depracate )

            

              

                

                  Para adicionar itens ao carrinho, você precisa fazer um POST para a URL definida na variável {{ cart_url }}, passando dois campos:


                
* product_id: ID do produto selecionado pelo usuário

                   * quantity: Quantidade de itens que o usuário deseja comprar


                 
Os campos product_id e quantity são obrigatório, mesmo que o produto tenha apenas uma variante.
                  Caso sua loja não permita que o usuário compre mais de um item, você deve informar este campo com o valor 1.
                  

                  
 Isso pode ser feito utilizando um formulário »

                  







                  
Em caso de o produto ter variantes » 

                  










                  
OU via AJAX »

                  





                  

                    Caso o produto possua variantes, você deve enviá-las junto com os parâmetros obrigatórios, da seguinte forma:
                  

                  

                    * attributes[cor]: Valor do atributo cor da variante

                    * attributes[tamanho]: Valor do atributo tamanho da variante

                   

                   
E assim para cada atributo configurado no admin.

                   

                      Além disso, você pode adicionar informações extras a cada item do carrinho.
                      Essas informações podem ser, por exemplo,
                      uma observação do cliente ou ainda uma imagem para ser impressa no produto.
                      Para enviar essas informações, basta criar os campos a seguinte forma »
                  

              

            

          

        

            


















<form action="{{ cart_url }}" method="POST">
  <input type="hidden" name="product_id" value="{{ product.id }}"/>
  <input type="number" name="quantity" min="1"/>
  <input type="submit" value="Adicionar"/>
</form>





<form action="{{ cart_url }}" method="POST">
  <input type="hidden" name="product_id" value="{{ product.id }}"/>
  <input type="number" name="quantity" min="1"/>
  <select name="attributes[tamanho]">
    <option value="P">P</option>
    <option value="M">M</option>
    <option value="G">G</option>
  </select>
  <input type="submit" value="Adicionar"/>
</form>
